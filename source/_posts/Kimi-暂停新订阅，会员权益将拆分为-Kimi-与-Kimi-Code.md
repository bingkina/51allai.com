> Kimi 暂停接受新会员订阅，并把算力优先分配给现有订阅用户。7 月 20 日起，Kimi 与 Kimi Code 会员权益分开：前者覆盖网页端、App 和 Work，后者面向编程工作流；老套餐可以继续使用和自动续费。
![Kimi 暂停新订阅并拆分 Kimi 与 Kimi Code 会员权益](https://images.51allai.com/blog/kimi-pauses-subscriptions-membership-split-cover-first_20260719_235441.png)

## Kimi 暂停接受新订阅

Kimi 在 7 月 19 日宣布暂停接受新会员订阅。过去 48 小时的需求已接近现有算力容量上限，平台将计算资源优先分配给当前订阅用户，已经订阅的用户不受这次调整影响。

暂停是临时措施。Kimi 正在增加算力，并会分批重新开放订阅名额。

## 会员权益从 7 月 20 日起拆分

Kimi 会员与 Kimi Code 会员将覆盖不同的产品：Kimi 会员用于网页端、App 和 Kimi Work，Kimi Code 会员面向终端、IDE 等编程工作流。需要同时使用两类产品的用户，要分别选择对应订阅。

Kimi Code 权益此前包含在 Kimi 会员中，与 PPT、Agent 等功能共享月度额度。拆分后，新版 Kimi Code 套餐使用编程专属额度，不再被其他 Kimi 功能的消耗占用；新版套餐也不包含网页端和 App 的会员权益。

## Kimi Code 保留周额度和 5 小时窗口

新版 Kimi Code 套餐取消月总额度上限，但仍保留周额度和滚动 5 小时频率窗口。额度从订阅日开始每 7 天刷新，未使用部分不会累积；短时间请求过多时仍会触发限流。

这套调整把日常对话、文档和桌面办公的用量，与代码读取、文件编辑和命令执行等编程任务分开计算。用户可以在 Kimi Code 控制台查看剩余额度、频率限制、API Key 和已登录设备。

## 老套餐可以继续使用和续订

已经购买老套餐的用户不需要强制迁移，可以继续使用并自动续订。老套餐仍同时包含通用 Kimi 会员和 Kimi Code 权益，只是停止向新用户销售。

用户也可以主动切换到新版 Kimi Code 套餐。切换后新套餐立即生效，老套餐同时失效并关闭自动续费；老套餐剩余价值会按有效期折算退回，原有 Kimi Code 额度则按新套餐规则重新计算。
