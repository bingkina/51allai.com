> DeepSeek V4 正式版被列入 7 月中旬上线窗口，API 计费可能从单一价格切到峰谷价。Pro 版输出在平时为 6 元/百万 tokens，高峰时段为 12 元；Flash 输出从 2 元升到 4 元。公开页面尚未同步这组调整，现阶段只能按 API 用户通知处理。
![DeepSeek V4 API 计费调整说明](https://images.51allai.com/blog/v4-spec_20260425_231923.png)

## 正式版时间窗口落在 7 月中旬

DeepSeek V4 预览版已经跑了两个多月。新的 API 用户通知给出下一步安排：V4 正式版计划在 2026 年 7 月中旬上线，更新方向包括功能优化和性能提升，但没有给出新的 benchmark、上下文窗口变化或模型架构细节。

这类信息不能写成完整发布。更准确的判断是：DeepSeek 已经开始给 API 用户做正式版切换前的计费预告，真正的模型参数、能力差异和接口兼容性，还要等公开文档和开发者实测。

## Pro 和 Flash 都引入高峰价

这次最实的变化不是模型名字，而是计费方式。通知里的价格表把每个模型拆成“平时价格”和“高峰时段价格”，高峰时段为北京时间每天 9:00-12:00、14:00-18:00。

`deepseek-v4-pro` 的新计费如下：

| 计费项 | 平时价格 | 高峰时段价格 |
| --- | ---: | ---: |
| 百万 tokens 输入（缓存命中） | 0.025 元 | 0.05 元 |
| 百万 tokens 输入（缓存未命中） | 3 元 | 6 元 |
| 百万 tokens 输出 | 6 元 | 12 元 |

`deepseek-v4-flash` 的新计费如下：

| 计费项 | 平时价格 | 高峰时段价格 |
| --- | ---: | ---: |
| 百万 tokens 输入（缓存命中） | 0.02 元 | 0.04 元 |
| 百万 tokens 输入（缓存未命中） | 1 元 | 2 元 |
| 百万 tokens 输出 | 2 元 | 4 元 |

峰谷差都是 2 倍。对轻量聊天类应用，缓存命中输入价格变化的体感不强；对 Agent、代码生成、长文生成这类输出 token 占比高的任务，高峰输出价会直接进入成本模型。

## 价格信号比涨跌本身更重要

V4-Pro 之前已经从限时折扣改成长期低价，开发者可以按 3 元输入、6 元输出做预算。新的通知如果落地，低价不会消失，但只属于非高峰时段。

这不是单纯涨价，更像容量调度。DeepSeek 把白天工作时段的高并发需求单独标价，夜间和非峰值时段继续保留低成本入口。对批处理、离线总结、知识库刷新、自动评测任务，迁移到非高峰时段可以保持原成本。对实时客服、IDE 编码助手、会议纪要这类不能错峰的产品，账单要按高峰价重新测算。

缓存命中价仍然低。Pro 的缓存命中输入即使在高峰时段也是 0.05 元/百万 tokens，Flash 是 0.04 元/百万 tokens。真正需要盯的是缓存未命中输入和输出。长链路 Agent 如果上下文复用率低，高峰期每轮调用的有效成本会放大。
